# 경제 분석 및 컴팩션 포렌식

[← 목차](README.md)

## 개요

Economic Usage 섹션은 세션 비용 역학, 컴팩션 영향 및 쿼터 효율성에 대한 심층 분석을 제공합니다. 표준 분석에서 드러나지 않는 숨겨진 비용을 밝힙니다: 점진적 리빌드 페널티, 쿼터 번 이상 및 장기 세션의 실제 비용.

## Cache Explosion — 턴당 비용

턴별 토큰 소비를 존 분석과 함께 보여주는 주요 차트입니다.

### 상단 그리드 (Scatter)

각 점은 하나의 API 턴을 나타냅니다. 총 비용 = `input + output + cache_read + cache_creation`.

- **녹색 존** (≤ 1.5× 베이스라인): Warmup — 캐시 구축 중, 비용 낮음
- **노란색 존** (≤ 3× 베이스라인): Linear — 안정적 성장
- **빨간색 존** (> 3× 베이스라인): Drain — 이차 성장 지배
- **보라색 점**: 컴팩션 이벤트 (확대)

오버레이 라인:

- **이차 피팅** (노란색 점선): `a·t² + b·t + c` 턴당 비용에 대한 최소제곱법
- **컨텍스트 크기** (파란색 점선): 캐시 건강 비율 `cache_read / (cache_read + cache_creation)`
- **비용 팩터** (분홍색): `cost[t] / cost[0]` — 첫 번째 턴 대비 승수

베이스라인 = 처음 50턴의 중앙값.

### 하단 그리드 — 토글

차트 아래 토글 버튼으로 두 가지 모드 접근:

#### 컨텍스트 손실 (기본)

컴팩션 이벤트로 인한 누적 토큰 손실을 보여주는 스텝 라인. 각 컴팩션 마커 표시:

- 토큰 드롭량과 백분율 (예: "Lost 160.8K (77%)")
- 각 컴팩션 위치에 수직선

세션에 컴팩션 이벤트가 있을 때만 표시됩니다.

#### 누적

턴 1에서 세션 끝까지 두 라인:

- **회색 라인 (실제)**: 턴별 실제 `cache_read` — 컴팩션 이벤트에서 드롭이 있는 톱니 패턴
- **녹색 라인 (Envelope)**: 컴팩션 드롭 없이 `cache_read`가 어땠을지, **M_real**로 가중
- **빨간색 영역**: 두 라인 사이의 격차 = 실제 리빌드 오버헤드
- **보라색 배지**: 각 컴팩션 이벤트에 `Real_Cost (M_real×)` 표시
- **존 영역**: Safe (녹색), Linear (노란색), Drain (빨간색) — 턴별 피팅 임계값 기반

axisPointer가 상단과 하단 그리드 간에 동기화됩니다.

## 점진적 컴팩션 페널티 (M_real)

**실증적 발견 (UDAA-29):** 컴팩션 손실은 점진적으로 처벌됩니다. 컴팩션 이벤트의 실제 비용은 단순한 토큰 드롭이 아닙니다 — 현재 (더 높은) 가격 수준에서의 리빌드 비용을 포함합니다.

### 공식

```text
Real_Cost = Drop × (1 + M(t))

M(t)     = f(t) / f_avg(t)
f(t)     = a·t² + b·t + c           위치 t에서의 현재 턴당 비용
f_avg(t) = a·t²/3 + b·t/2 + c       턴 0..t의 평균 비용 (적분 / t)
```

### 왜 (1 + M(t))인가

- **1×** = 드롭 자체 (손실된 토큰)
- **M(t)×** = 리빌드 비용 (원래 더 저렴하게 구축된 것을 현재 가격으로 재구축)

### 수렴

| 세션 위치 | M_real | 의미 |
| --- | --- | --- |
| 초기 (Safe 존) | ~2.0× | 리빌드 비용이 원래 구축과 거의 같음 |
| 중간 (Linear 존) | ~2.5× | 리빌드가 눈에 띄게 더 비쌈 |
| 후기 (Drain 존) | ~2.7× | 리빌드가 상당히 더 비쌈 |
| 이론적 최대값 | →4.0× | 리빌드가 원래의 3배 (1 + 3) |

### 이중 효과

드롭 자체가 턴마다 커지고 (cache_read 증가, 컴팩션 하한은 ~48K 유지), 승수도 증가합니다. 늦게 떨어짐 = 더 깊이 떨어짐 × 더 비싼 재구축.

### 핵심 특성

- M_real은 **절대 정적이지 않음** — 턴 위치, 세션 베이스라인, 곡선 형태 (a, b, c)에 의존
- 각 세션마다 고유한 이차 피팅 계수
- 음수 `a` (오목 성장)는 M_real을 약 2.6×으로 제한
- 양수 `a` (진정한 이차 폭발)는 M_real을 4.0× 방향으로 밀어냄

### 실증 데이터 (2026-04-12)

**세션 9afe9ab1** (927턴, 9 컴팩션, 모두 Drain 존):

- M_real 범위: 2.50–2.67×
- Raw 드롭: 2.9M 토큰 → 실제 비용: 7.7M 토큰
- 오버헤드: 세션의 2.8%

**세션 c2f4700e** (736턴, 12 컴팩션, Linear→Drain):

- M_real 범위: 2.43–2.73×
- Raw 드롭: 2.4M 토큰 → 실제 비용: 6.4M 토큰
- 오버헤드: 세션의 4.2%

## 컴팩션 시 Q5 페널티 (UDAA-30)

**실증적 발견:** 컴팩션 이벤트 후 Q5 implied divisor가 일시적으로 중앙값의 20–35%로 하락합니다. 캐시 리빌드 단계에서 Anthropic이 토큰당 3–5배 더 많은 쿼터를 청구한다는 의미입니다.

| 캐시 상태 | 평균 Q5 디바이저 | 토큰당 비용 |
| --- | --- | --- |
| 웜 (Health > 80%) | ~25 (중앙값) | 정상 |
| 컴팩션 후 (Health < 30%) | 4–9 | 3–5× 더 비쌈 |

- 컴팩션 이벤트 근처 Q5 비용의 **57%가 페널티**
- 회복 시간: 디바이저 정상화까지 30–150초
- 총 Q5 초과: 일일 +1.28% (~3.8분 세션 시간 손실)

### 두 가지 독립적 페널티 메커니즘

컴팩션은 두 개의 별도 레이어에서 페널티를 발생시킵니다:

1. **토큰 레이어**: M_real = 2.4–2.7× (리빌드에 더 많은 토큰 소비)
2. **쿼터 레이어**: Q5 디바이저 붕괴 = 리빌드 중 토큰당 3–5× 더 많은 쿼터

이들은 예산 영향에서 곱셈이 아닌 **덧셈** 관계입니다.

## 리스타트 경제학

출력 토큰당 한계 비용에 기반한 최적 세션 리스타트 간격 분석.

### 문제

세션이 성장함에 따라 생산적 출력 대 총 비용 비율이 악화됩니다:

| 턴 범위 | 평균 비용/턴 | 평균 출력/턴 | 효율성 |
| --- | --- | --- | --- |
| 0–99 | 92K | 209 | 0.23% |
| 400–499 | 298K | 117 | 0.04% |
| 900–926 | 481K | 135 | 0.03% |

턴 900에서 ~200 토큰의 출력을 위해 ~500K 토큰을 지불합니다. 99.7%가 컨텍스트 유지입니다.

### 손익분기점 분석

| 전략 | 리스타트 | 총 비용 | vs 리스타트 없음 | 절약 |
| --- | --- | --- | --- | --- |
| 리스타트 없음 (927t) | 0 | 270M | 기준 | — |
| 100턴마다 | 14 | 119M | 0.44× | **56%** |
| 150턴마다 | 7 | 128M | 0.47× | **53%** |
| 250턴마다 | 3 | 132M | 0.49× | **51%** |
| 300턴마다 | 3 | 174M | 0.65× | **35%** |
| 700턴마다 | 1 | 336M | 1.25× | **−25%** |

100–150턴마다 리스타트하면 워밍업 오버헤드를 포함해도 53–56% 절약.

**손익분기점:** 턴 ~183 — 이후 모든 추가 턴은 리스타트 후보다 비용이 더 큽니다.

## Extract-Cache (성능)

세션 턴 계산을 위한 사전 추출 캐시:

- **`scripts/extract-cache.js`** — 관련 JSONL 레코드 추출 (~150바이트 vs 5–50 KB)
- 추출 중 신호 감지 실행
- mtime+size를 포함한 파일 매니페스트로 증분 동기화

| 시나리오 | 캐시 없이 | 캐시 있음 | 속도 향상 |
| --- | --- | --- | --- |
| Extract-Cache 빌드 | 95s | 169ms | 560× |
| 세션 턴 | 38.5s | 7ms | 5,500× |

## Budget Drain — 일일 세션 영향

일일 쿼터 소비의 세션 간 뷰. 턴당 잔여 쿼터 백분율 표시:

- 세션별 그래디언트 채움 (녹색→빨간색)
- 세션 경계 마커 (파란색 = 정상, 빨간색 = 강제 리스타트)
- 강제 리스타트 시 리빌드 비용 배지
- 캐시 건강 오버레이 (warm/cooling/cold)
- 컴팩션 이벤트 마커
- Q5 실제 vs Q5 이상 오버레이 (프록시 데이터 필요)
