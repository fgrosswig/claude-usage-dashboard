# 인텔리전스 섹션 및 메트릭스 엔진

[← 목차](README.md)

> **상태:** 초기 구현. 가중치, 점수 공식 및 임계값은 잠정적이며 실제 사용 데이터를 기반으로 검증 및 보정이 필요합니다.

## 개요

인텔리전스 섹션은 API 사용에 대한 예측 메트릭과 자동 분석을 제공합니다. **메트릭스 엔진** (`public/js/metrics-engine.js`) — 7개의 독립적인 순수 함수 모듈로 구동됩니다.

## 메트릭스 엔진 모듈

### 1. 포화 점수 (Saturation Score)

4가지 요소의 가중 복합 점수 (0-100):

| 요소 | 가중치 |
|------|--------|
| 지연 시간 | 30% |
| 오류율 | 20% |
| 캐시 미스 | 15% |
| 쿼터 사용률 | 35% |

높은 값 = 시스템에 부하.

### 2. 쿼터 ETA

쿼터 소진까지 예상 잔여 시간. 계산:

- **기본**: `q5_samples`(5시간 윈도우 프록시 데이터)의 소진율
- **폴백**: 활성 시간의 평균

### 3. EWMA (지수 가중 이동 평균)

- `ewma(prevValue, newValue, alpha)` — 단일 값
- `ewmaArray(values, alpha)` — 배열 평활화
- 지연 시간 및 오류율의 추세 감지에 사용

### 4. 건강 점수 (Health Score)

역전된 복합 점수 (0-100, 높을수록 양호):

| 차원 | 가중치 |
|------|--------|
| 신뢰성 | 35% |
| 용량 | 30% |
| 효율성 | 35% |

### 5. 근본 원인 탐지

현재 상태에 가장 많이 기여하는 상위 5개 요소, 평균 대비 델타 백분율 포함.

### 6. 서술적 요약

색상 점이 있는 사람이 읽을 수 있는 상태 문구:
- 초록: 정상
- 노랑: 주의 필요
- 빨강: 위험

### 7. 계절성 기준선

시간대별 평균 요청 수와 지연 시간 (0-23시) — 일반적인 일일 패턴 표시.

## 인텔리전스 섹션 UI

섹션에 표시되는 항목:

- **3개 KPI 카드**: 포화 점수, 건강 점수, 쿼터 ETA
- **서술 박스**: 자동 생성된 상태 설명
- **근본 원인 패널**: 백분율 편차가 있는 상위 요소
- **계절성 차트**: ECharts 막대 차트 (시간당 요청)

## 데이터 소스

- **프록시 데이터**: 지연 시간, 오류율, 캐시 적중률, 쿼터 샘플
- **JSONL 데이터**: 시간당 요청, 토큰 소비
- **계산된 값**: 모든 점수는 클라이언트 측에서 실시간으로 계산

## Extract-Cache (성능)

세션 턴 계산(경제적 사용)을 위해 사전 추출 캐시가 도입되었습니다:

- **`scripts/extract-cache.js`** — 관련 JSONL 레코드 추출 (레코드당 5-50KB 대신 ~150바이트)
- 추출 중 신호 감지 (classifySignals, scanLineHitLimit) 실행
- 증분 동기화를 위한 mtime+size 파일 매니페스트

### 성능

| 시나리오 | 캐시 없음 | 캐시 사용 | 배수 |
|----------|----------|----------|------|
| Extract-Cache 빌드 | 95초 | 169ms | 560배 |
| Session-Turns | 38.5초 | 7ms | 5500배 |

`session-turns-core.js`는 빠른 경로를 위해 `pass1FromExtractCache()`와 `buildSessionTurnsFromCache()`를 사용합니다.
